What is Elixir?

What is Erlang?

- designed for fault tolerance
- distributed computing support
- multi-threaded/multi-core concurrency
- code hot-swapping
- integrated release workflow (trivially deployable)

Elixir is all this and more!

- great, mature tooling: Mix, Hex
- simple, friendly syntax
- polymorphism via protocols
- cleaned up standard library
- many interesting libraries and frameworks to choose from

Is Elixir production-ready?

Yes!

- Erlang and Erlang VM are ~30 years old
- a mighty VM forged in the heat of battle,
   rivaling JVM and CLR with ease
- same goes for most libraries

Also...

- stability, "nine nines" uptime
- speed, compared to non-jitted Python or Ruby
- built-in distribution and scalability support
- built-in production-grade logging
- production-tuned tools like trace, profiler, process manager...

Last but not least

- deploying releases with 0 downtime
- even across clusters and multiple nodes
- which are discovered and connected automatically, BTW
- deployment is as easy as with Go: just copy an
   executable where you want it, done
- works well with Jenkins, AWS and other popular solutions

Problems?

Not really, but...

- much less libraries to choose from compared to Python
  (but it's easy to call Python code directly with ErlPort)
- actor-based, shared-nothing concurrency is unfamiliar
- heavily biased towards Functional Programming
- as clusters grow in size, they need managing
- may require tuning VM for optimal performace

So?

- your next project should be written in Elixir!
- especially if it's a DB backed API server
- with Phoenix, even standard dynamic pages are feasible
- it's even quite alright for writing quick scripts
- in general all the backend infrastructre may be written
   in Elixir without any problems
- let's go do some coding in Elixir!
